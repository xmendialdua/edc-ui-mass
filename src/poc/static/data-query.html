🎯 Objetivo y Flujo de Consumer Mode ▼

🎯 Objetivo

Ejecutar el flujo completo de consumo de datos: descubrir assets en el catálogo de MASS, negociar contratos y realizar transferencias de datos.

📖 Flujo de Consumer Mode

1. Descubrimiento (Catalog Discovery)

  • IKLN consulta el catálogo de MASS mediante el protocolo DSP
  • Se muestran los datasets disponibles y sus assets asociados
  • Cada asset incluye sus políticas y metadatos

2. Negociación (Contract Negotiation)

  • Se selecciona un asset y se inicia la negociación de contrato
  • MASS evalúa las políticas y responde con el estado de la negociación
  • Estados posibles: REQUESTED → NEGOTIATING → AGREED → FINALIZED
  • Si falla, se muestra el motivo del error

3. Transferencia (Data Transfer)

  • Con un contrato finalizado, se inicia la transferencia de datos
  • MASS Data Plane proporciona acceso al recurso
  • Se monitorea el progreso de la transferencia

⏱️ Nota

Los procesos de negociación y transferencia son asíncronos y pueden tardar varios segundos.
